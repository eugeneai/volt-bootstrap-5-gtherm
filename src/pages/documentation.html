Instruction for Geotherm program usage

We are pleased to announce the launch of the Geoterm model calculation application, which allows one to calculate the heat flow and lithospheric mantle thickness. The software also provides a range of visualization options, generating custom graphs capable to be an editable graphics material of scientific publications. We hope that you will find the Geoterm application useful in your work or research, and we look forward to hearing your feedback.

If you have any questions or would like to learn more about the software, please contact us!

Step 1. Registration

To create a personal account and store all your data and calculations, you will need to register with the website's login/register page. The registration process begins by clicking the button located in the upper right-hand corner of the page, and then proceeds by clicking on the "Sign Up!" button (indicated in red in the image below).

Next, you will be directed to the registration form, where you will enter information such as a brief nickname, surname, first name, organization, email address, and password. The email confirmation does not work now and will not in the nearest future.

Step 2. Upload data

After registration has been completed, you will be directed to the Dashboard page. If this has not occurred, please click on the relevant button in the menu on the left-hand side. To upload the Excel file containing the data to be processed, click the “Select file” (1) button and then "+New Project" (2).

Figure 1. Uploading data site page

Figure 2. Creating new project

The application can process CSV- and XLSX-files only. For XLSX-files, the uploader recognizes pages, producing a set of data tables (hereafter datasets).

Few words about the input data table format assumptions. The application checks if the first five columns of the table contain nonempty data, as well as the first ten rows. If this area does not contaion non-empty cells, the table (e.g. Excel page) is ignored at all.

Step 3. Naming columns

After your datasets has been added to the list, choose one from the list by clicking its title with the left mouse button.

You will be directed to the "Measurement" page, where you have to assign application-understandable (standardized) column names to your dataset columns. To do so, click on the button indicated in figure (1) and select the appropriate name and units of measurement (2). Once you have completed this step, click on button 3 to save the assignments. The assignments are associated with this one dataset only. For all new similar datasets the assignment process have to be repeated.

Figure 1. Naming a column (two mouse left-clicks)

Application could be told to ignore a column at all by a corresponding assignment. Columns with string data are to be designated as "Name" columns, they are also ignored.

The assignment step could be skipped, if input table columns are named according this rule.

Step 4. Changing the paleogeotherm model parameters

After naming the columns, switch from the "Measurement" tab to the "Model" tab.

In this page, the model parameters for the crust and lithospheric mantle geometry are set by default for a most common case. To improve the accuracy of the paleogeotherm they can be altered. Standard parameters for the software were derived from a study by Hasterok and Chapman (2011). To obtain an optimized paleogeotherm vslue q0 (the surface heatflow) and a more accurate representation of the thermal characteristics of the lithospheric mantle, click the slider (1). Clicking "Save" (2) or "Calculate" (3) finalizes model data adjustenments, saving them on the server for the chosen dataset. Calculate button starts paleogeotherm construction according to the model setup.

Step 5. Presentation of the computation results in graphical form

After calculating the data, navigate to the "Graphs" tab.

The tab presents the results of the paleogeothermal calculation. The first figure, which is usually located on the left side, displays the family of geothermal lines within the range specified in the previous step, along with the P-T data points.

In the optimization option is on two more figures are generated.

The first one looking like horse saddle, illustrates the optimized value of the q paraveter and its misfit value.

The second one is paleogeothermal model for the assessed optimized value.

All the fugures can be downloaded using a corresponding "Download" button in SVG vector format, which is editable with e.g. Inkscape program and imported as vector graphics in reasonable modern office package.

References

  1. Derrick & Chapman, D.S. (2011). Heat production and geotherms for the continental lithosphere. Earth and Planetary Science Letters. 307. 59-70. doi:10.1016/j.epsl.2011.04.034.
